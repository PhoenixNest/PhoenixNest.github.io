背景

游戏介绍

《星际争霸Ⅱ》（StarCraftⅡ）是由暴雪娱乐在2010年7月27日推出的一款即时战略游戏，是《星际争霸》系列的第二部作品。游戏以三部曲的形式推出，即《星际争霸Ⅱ：自由之翼》（Wings of Liberty）、《星际争霸Ⅱ：虫群之心》（Heart of the Swarm）和《星际争霸Ⅱ：虚空之遗》（Legacy of the Void）。游戏结构类似于《星际争霸》，主要的游戏技巧着重在资源上，玩家用采集的资源建造不同的建筑、军队并进行升级 。

《星际争霸》有着专业的职业赛事与选手选拔机制，其中不乏进入过早期有着“电子竞技奥运会”之称的WCG（World Cyber Games），官方赛事有着诸如：暴雪WCS（World Championship Series）世界锦标赛，ESL（Electronic Sports League）电子竞技联盟与GSL（Global StarCraftⅡ League）全球《星际争霸Ⅱ》联赛等赛事。

DeepMind • AlphaStar

2016年，继AlphaGo在韩国首尔击败顶尖棋手李世石后，DeepMind的下一个目标战场来到了《星际争霸Ⅱ》，尝试在战略游戏领域挑战顶尖人类选手。次年，DeepMind开始在欧服天梯上放出自己开发的人工智能AI，开始在PVP实战中训练AI。

时间转到2019年，在DeepMind的人工智能登顶星际争霸最高天梯级别（宗师）后，DeepMind释出了AlphaStar，一个专精于星灵（神族）的人工智能AI，并于1月首次公开挑战来自Team Liquid的职业选手Mana与TLO，结局也如DeepMind所期愿，AlphaStar仅仅输了两局，便击败了这两位欧洲职业选手。

籍此，AlphaStar的胜利将AI的可能性再度拔高，并在《Nature》发表了有关AlphaStar的论文。

建造函数

星灵枢纽（基地）

星灵枢纽，作为神族主基地的大型建筑，这种宏伟的建筑为神族提供初期补给（10人口）与基础采集单位建造权限，同时为采集单位提供资源处理集中点，在游戏开始时玩家在出生点处会拥有一座基地与12个采集单位。

随着游戏进展与战术选择，单矿战术与多矿战术决定了整场对局中你所拥有的基地数量，若选择常规运营开局，则通常需要扩张大量分基地对战局资源进行高效采集，同时构造纵深的防御阵线（前线基地与后方主阵地）。

键盘快捷键：BN

水晶塔

水晶塔是神族重要的补给建筑，不仅为神族提供人口，且为神族建筑群供能，并提供折跃立场以供传送门传送前线部队。

通常建造水晶塔的时间点有两个，第一个时间点是在游戏初期生产第1个农民后，生产第2个农民前，需建造第一座水晶塔对人口进行补充；后续时间点通常为剩余人口数小于5时建造。

键盘快捷键为：BE

async def build_pylons(self):
    # 使水晶塔不占据矿区影响采集效率
    build_position = self.structures(UnitTypeId.NEXUS).first.position.towards(self.game_info.map_center, 5)
        if self.supply_left < 5 and not self.already_pending(UnitTypeId.PYLON):
            if self.structures(UnitTypeId.NEXUS).ready:
                if self.can_afford(UnitTypeId.PYLON):
                    await self.build(UnitTypeId.PYLON, near=build_position)

吸纳仓

吸纳仓是神族专有的采集瓦斯气体的建筑，仅能放置在游戏中基地两旁的瓦斯气矿上并覆盖气矿，吸纳仓每次生产3单位的瓦斯产量，每一座吸纳仓建议的采集农民数量为3个。

神族在游戏初期的开气矿时间为通常为水晶塔建造完成后（大约15个农民时间点左右）。

键盘快捷键：BA

async def build_gas_collector(self):
    for nexus in self.structures(UnitTypeId.NEXUS).ready:
        vaspenes = self.vespene_geyser.closer_than(15, nexus)
        for vaspene in vaspenes:
            if not self.can_afford(UnitTypeId.ASSIMILATOR):
                break

            worker = self.select_build_worker(vaspene.position)
            if worker is None:
                break

            if not self.structures(UnitTypeId.ASSIMILATOR).closer_than(1.0, vaspene).exists:
                worker.build(UnitTypeId.ASSIMILATOR, vaspene)

传送门

作为神族前期能建造的一级军事建筑，传送门为神族提供基础地面单位

  • 前期基础战斗单位：狂热者、追猎者、机械哨兵与使徒

  • 中后期进阶战斗单位：高阶圣堂武士与黑暗圣堂武士

键盘快捷键：BG

传送门通常为偶数增长，常见的传送门增长系数为2BG，4BG（最著名的神族前期一波战术打法），8BG，10BG不等（BG为传送门在神族建筑列表中的键盘快捷键，即：按下B键后按G键快速选中传送门）

async def build_gateway(self):
    pylon = self.structures(UnitTypeId.PYLON).ready.random
    if self.can_afford(UnitTypeId.GATEWAY) and not self.already_pending(UnitTypeId.GATEWAY):
        await self.build(UnitTypeId.GATEWAY, near=pylon)

控制芯核

控制芯核是神族由前期过渡至中后期所必需要建造的科技型建筑，解锁神族高阶军事建筑（如：光影议会、机械台、星门、舰队航标等）与中高级战斗单位（如机械哨兵、追猎者），并为神族提供二级传送门科技（折跃）与空军攻防升级。

控制芯核通常只需要一座，但伴随游戏进展至中后期，需要对现有部队进行转型（空军与高级地面单位混编）与升级（空军攻防）时则需要增加至多座，其中不免包括中期可能因为敌方骚扰与渗透而导致的控制芯核被摧毁而需重建等不确定因素。

键盘快捷键：BY

async def build_cybernetic_core(self):
    if self.structures(UnitTypeId.CYBERNETICSCORE).ready.amount <= 2:
        if self.can_afford(UnitTypeId.CYBERNETICSCORE) and not self.already_pending(UnitTypeId.CYBERNETICSCORE):
            await self.build(UnitTypeId.CYBERNETICSCORE, near=self.structures(UnitTypeId.PYLON).ready.random)

锻炉

作为神族地面单位主要的升级建筑之一，锻炉为神族地面单位提供攻击与护甲升级，并提供全神族单位的护盾升级。

对于攻击与护甲升级，通常简称为GF（攻防），1G则代表升级了1级攻击，针对攻击力的升级等式，有如下例子：

针对神族近战单位狂热者的攻击升级，将会为期提供1点攻击加成，其武器由原本8x2升级为9x2，这对于单个单位可能没有显著的提升，但对于中后期的全战局将会起到质的飞跃，优先升级攻防的一方将在升级完成后的1分钟内采取主战权，通常称为Timing。

由8个狂热者组成的骚扰小队能在中后期的分兵作战中起到奇袭作用，包括从侧路突破敌军，对敌军阵后进行骚扰迫使敌方主力回防或者切断后续增援等战术操作。

缎炉通常需要数量都是两座，但也需要视对局种族而进行建造，如神族内战（PVP）时，对于攻防的偏重经常是先升攻再升防，强调攻击升级，此时也可只放置一座缎炉，后续增至两缎炉同时升级。

随着时间往大后期推移，通常双方均能达到满级攻防（神族除攻防升级外还有护盾升级），此时将对双方操作容错率有极大考察，因为一个小失误而导致的局部失利将会如滚雪球式对整场比赛造成难以挽回的影响。

键盘快捷键：BF

护盾充能器

作为神族主要的两大防御建筑之一，护盾充能器能在短时间内为神族单位与建筑恢复大量护盾数值，也称为护盾电池。

神族无法像人族单位可以被治疗与维修，也不像虫族单位能自动恢复生命值，但神族拥有前两个种族所没有的护盾科技，由于护盾的受击优先级大于护甲，在生命值受损前，护盾会吸收大量伤害。

护盾通常在脱离战斗后会自行恢复，也可藉由神族专有的护盾充能器进行充能。

护盾在某些特殊战术中也充当进攻角色，如在神族与人族（PVT）的比赛中，通常也有使用前置电池掩护并采取星门一波的战术。

键盘快捷键：BB

光子炮台

作为神族主要的两大防御建筑之一，光子炮台可侦测范围内隐形单位（如神族黑暗圣堂武士、人族特种作战单位幽灵、虫族埋地单位等），通常作为中后期主要静态防御建筑。

在矿区设置光子炮台是前中期防止敌军骚扰的有效手段，一个光子炮台能对敌方骚扰单位起到高度威慑作用（如神族前期空军单位先知对矿区的干扰），迫使其放弃骚扰并撤离。

在分矿处设置光子炮台是有效的中后期防止敌方分兵骚扰的手段，通过与护盾充能器配合，多个光子炮台能有效牵制敌方主力部队与防止敌方分兵偷袭。

在针对人族的多线空投与虫族的中期飞龙骚扰中，光子炮塔也能在增援部队到达前削弱敌方单位数量。

键盘快捷键：BC

光影议会

作为控制芯核建造完毕后，神族首个解锁的二阶科技建筑，光影议会为神族传送门单位：狂热者、追猎者与使徒提供升级项。

  • 冲锋：使狂热者能冲刺并拦截敌方单位，该项升级还将为狂热者提供移速加成，冷却时间7秒。

  • 闪现：使追猎者能闪现至目标位置（该位置必须可见且在范围内），冷却时间7秒。

  • 共鸣之刃：为使徒提供百分之45的攻速加成。

键盘快捷键：VC

圣堂文库

三级科技建筑，解锁神族高阶圣堂武士并为其提供升级项。

黑暗圣坛

三级科技建筑，解锁神族黑暗圣堂武士并为其提供升级项。

  • 暗影潜行：使黑暗圣堂武士能快速切入目标位置，常用于对关键目标（如人族攻城坦克）进行精准斩杀或对矿区进行骚扰，黑暗圣堂武士单次攻击能直接击杀任何采集单位，该特性常被用来进行前期针对性经济压制（PVP或PVZ前期没有有效反隐手段时有奇效）与中期骚扰。

async def scout(self):
    for expansion_location in self.expansion_locations_list:
        distance_to_enemy_start = expansion_location.distance_to(self.enemy_start_locations[0])
        self.expand_dis_dir[distance_to_enemy_start] = expansion_location
    self.ordered_exp_distances = sorted(k for k in self.expand_dis_dir)
    existing_ids = [unit.tag for unit in self.units]

    # 当OB被对方摧毁时从列表中去除
    to_be_removed = []
    for noted_scout in self.scouts_and_spots:
        if noted_scout not in existing_ids:
            to_be_removed.append(noted_scout)
    for scout in to_be_removed:
        del self.scouts_and_spots[scout]
    if self.structures(UnitTypeId.ROBOTICSFACILITY).ready.amount == 0:
        unit_type = UnitTypeId.PROBE
        unit_limit = 1
    else:
        unit_type = UnitTypeId.OBSERVER
        unit_limit = game_config.total_ob_num
    assign_scout = True

    # 前期农民探路
    if unit_type == UnitTypeId.PROBE:
        for unit in self.units(UnitTypeId.PROBE):
            if unit.tag in self.scouts_and_spots:
                assign_scout = False
    if assign_scout:
        if self.units(unit_type).idle.amount > 0:
            for obs in self.units(unit_type).idle[:unit_limit]:
                if obs.tag not in self.scouts_and_spots:
                    for dist in self.ordered_exp_distances:
                        try:
                            location = next(value for key, value in self.expand_dis_dir.items() if key == dist)
                            # DICT {UNIT_ID:LOCATION}
                            active_locations = [self.scouts_and_spots[k] for k in self.scouts_and_spots]
                            if location not in active_locations:
                                if unit_type == UnitTypeId.PROBE:
                                    for unit in self.units(UnitTypeId.PROBE):
                                        if unit.tag in self.scouts_and_spots:
                                            continue
                                self.do(obs.move(location))
                                self.scouts_and_spots[obs.tag] = location
                                break
                        except Exception as exception:
                            print("EXCEPTION | Scout: ", str(exception))

    for obs in self.units(unit_type):
        if obs.tag in self.scouts_and_spots:
            if obs in [probe for probe in self.units(UnitTypeId.PROBE)]:
                self.do(obs.move(self.random_location_variance(self.scouts_and_spots[obs.tag])))

# 随机地点
def random_location_variance(self, location):
    x = location[0]
    y = location[1]

    x += random.randrange(-5, 5)
    y += random.randrange(-5, 5)

    if x < 0:
        print("x below")
        x = 0
    if y < 0:
        print("y below")
        y = 0
    if x > self.game_info.map_size[0]:
        print("x above")
        x = self.game_info.map_size[0]
    if y > self.game_info.map_size[1]:
        print("y above")
        y = self.game_info.map_size[1]
    go_to = position.Point2(position.Pointlike((x, y)))
    return go_to
