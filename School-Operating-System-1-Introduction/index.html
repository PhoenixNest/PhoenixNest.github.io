引言

本篇整理了我学习操作系统时的一些笔记。
所用教学平台：BiliBili

定义

控制和管理计算机系统资源，合理调度各类资源，提高系统效率，方便用户使用的系统软件。核心如下：

  • 控制和管理计算机系统的硬件、软件资源。

  • 合理调度各类资源，提高系统效率和吞吐量。

  • 方便用户使用。

  • 硬件系统之上的第一层软件。

作用

  • 用户和计算机间的接口：使用户能够更方便、更快捷、更安全、更可靠地操纵计算机硬件和运行程序。

  • 资源管理者：管理计算机系统资源。

目标

（1）方便性

无操作系统的计算机，用户只能直接使用硬件，必须用机器语言变成，自己管理内存、外设等硬件，既不方便、难度又极大。配置操作系统后，用户只需利用操作系统提供的接口或命令就可方便使用计算机。

**（2）有效性

推动操作系统发展的最主要动力是提高资源利用率。

  • 提高资源利用率

  • 提高系统吞吐量

注意：方便性和有效性是操作系统最重要的两个目标。

（3）可拓展性

和操作系统结构密切相关，并推动操作系统结构的发展。

可拓展性包括以下两个方面内容：

  • 方便操作系统增添新功能。

  • 对原有功能进行修改。

（4）开放性

遵循世界标准规范，是当前计算机技术的核心问题，也是衡量新系统是否能广泛应用的重要因素。

开放性主要包含以下两方面内容：

  • 兼容性：方便互连。

  • 遵循标准：按照同样的标准设计和生产。

发展动力

  • 器件的不断更新迭代。

  • 不断涌现的应用需求。

  • 计算机体系结构的不断发展。

  • 不断提高计算机资源利用率。

  • 方便用户：提供更直观、更方便的接口。

手工操作阶段

  • 优点：处理速度快。

  • 缺点：用户独占全机、人机速度矛盾导致资源利用率极低。

批处理阶段 - 单道批处理阶段

引入脱机输入/输出技术（用磁带完成），并监督程序（操作系统雏形）负责控制作业的输入、输出。

  • 优点：缓解了一定程度的人机速度矛盾，资源利用率有所提升。

  • 缺点：内存中仅能由一道程序运行，只有该程序运行结束之后才能调入下一道程序。CPU有大量的时间是在空闲等待 I/O 完成。资源利用率依旧很低。

批处理阶段 - 多道批处理阶段

每次往内存中输入多道程序。操作系统正式诞生，并引入了中断技术，由操作系统负责管理这些程序的运行。各个程序并发执行。

  • 优点：多道程序并发执行，共享计算机资源。资源利用率大幅提升，CPU和其他资源保持“忙碌”状态，系统吞吐量增大。

  • 缺点：用户响应时间长，没有人机交互功能（用户提交自己的作业之后就只能等待计算机处理完成，中间不能控制自己的作业执行）。

分时操作系统

计算机以时间片为单位轮流为各个用户/作业服务，各个用户可通过终端与计算机进行交互。

  • 优点：用户请求可以被即使响应，解决了人机交互问题。允许多个用户同时使用一台计算机，并且用户对计算机，并且用户对计算机的操作相互独立，感受不到别人的存在。

  • 缺点：不能优先处理一些紧急任务。操作系统对各个用户/作业都是完全公平的，循环地为每个用户/作业服务一个时间片，不区分任务的紧急性。

实时操作系统

  • 优点：能够优先响应一些紧急任务，某些紧急任务不需时间片排队。

  • 缺点：在实时操作系统的控制下，计算机系统接收到外部信号后及时处理，并且要在严格的时限内处理完事件。

实时操作系统的主要特点是及时性和可靠性。

网络操作系统

是伴随着计算机网络的发展而诞生的，能把网络中各个计算机有机地结合起来，实现数据传送等功能，实现网络中各种资源的共享（如：文件共享）和各台计算机之间的通信。（如：WindowsNT 就是一种典型的网络操作系统，网站服务器就可以使用）

分布式操作系统

主要特点是分布性和并行性。系统中的各台计算机地位相同，任何工作都可以分布在这些计算机上，由他们并行、协同完成这个任务。

个人计算机操作系统

如：Windows 10，macOS，Linux，方便个人使用。

特征

注意：
并发和共享互为存在条件。
没有并发和共享，就谈不上虚拟和异步，并发和共享是多用户操作系统最基本的两个特征。

（1）并发

两个或两个以上的程序在同一时间间隔同时处于执行状态，通过资源复用实现。这些事情宏观上是同时发生的，但微观上是交替发生的。

操作系统的并发性指计算机系统中同时存在多个运行着的程序。

一个单核处理机（CPU）同一时刻只能执行一个陈鼓型，因此操作系统会负责协调多个程序交替执行。

事实上，操作系统是伴随着“多道程序技术”而出现的。因此，操作系统和程序并发是一起诞生的。

（2）共享

资源复用。不同属性的资源的共享方式有如下：

  • 互斥共享：不同进程顺序访问，如打印机、磁带机，称为临界资源。系统中的某些资源，虽然可以提供给多个进程使用，但一个时间段内只允许一个进程访问该资源。

  • 同时访问：不同进程交替访问，如磁盘。系统中的某些资源，允许一个时间段内由多个进程“同时”对它们进行访问。

并发性指计算机系统中同时存在着多个运行着的程序。
共享性是指系统中的资源可供内存中多个并发执行的进程共同使用。

（3）虚拟

通过某种技术将一个物理实体变成若干个逻辑上的对应物体。主要包含以下两种方式：

  • 时分复用（如虚拟存储器技术）

用户占用某些设备后并不一定一直使用，可利用设备为某用户服务的空闲时间为其他用户服务。如：

多道程序设计：虚拟处理器。

虚拟设备：共享打印机，如 SPOOLing 系统。

特点：若 1 个物理设备 N 个用户时分复用，则每个用户使用的虚拟设备的平均速度不高于物理设备速度的 1/N 。

  • 空分复用（如虚拟处理器）

用户占用某些设备后并不一定全部使用，可利用某用户不使用的部分设备资源为其他用户服务。如：

虚拟存储器：外存模拟内存、

特点：若 1 个物理设备 N 个用户空分复用，则每个用户占用的虚拟设备空间不大于物理设备总空间的 1/N 。

（4）异步性

多道程序环境下，允许多个程序并发执行。

由于资源有限，进程的执行不是一贯到底的，而是走走停停，已不可预知的速度向前推进。

  • 执行顺序、执行进度不确定。

  • 运行环境相同，同一个作业经多次运行，都会获得完全相同的结果。

只有系统拥有并发性，才有可能导致异步性。

功能

（1）CPU/处理机管理

以进程为单位进行处理机分配。将对处理机的管理转化为对进程的管理和调度。

进程管理：进程控制

  • 为作业创建一个或多个进程，并为之分配必要的资源。

  • 撤销已结束进程，回收其占用的资源。

  • 控制进程运行过程的状态转换。

进程管理：进程同步（协调多个进程的进行）

  • 互斥：进程对临界资源的使用方式，常用互斥方法为锁机制。

  • 同步：进程之间通过相互合作完成共同任务，常用信号量机制。

进程管理：进程通信

相互合作的诸进程之间需要互换信息，即合作进程需要通信。

进程调度：分配资源

  • 作业调度：为部分后备作业（存在于外存，一般称为后备作业队列）分配资源、将其调入内存、创建进程、插入到就绪队列，等待进程调度。

  • 进程调度：为就绪队列的某进程分配处理器，设置运行现场、使其能够运行。

进程：待运行 -> 执行

-> 外存，后备作业队列（ n 个待运行进程）。 
-> 作业调度（从 n 个进程中挑选 m 个进程进入内存，其中 m < n）。 
-> 内存（m 个进程，进入就绪队列）。
-> 进程调度（从 m 个进程中挑选 1 个进程，为其分配处理机，1 个处理机只接 1 个进程，并设置运行现场）。
-> 进程进入执行态。

（2）存储器管理

为多道程序的运行提供良好运行环境，提高内存利用率，从逻辑上扩充内存空间。

内存分配：

  • 静态分配：装入作业时分配其所需全部存储空间，运行期间不可再申请存储空间，也不能移动。

  • 动态分配：装入作业时分配其所需的基本存储空间，运行期间可再申请存储空间，也可以移动。

内存保护：

通过设置界限寄存器实现，主要任务是保证各道程序之间互不干扰。

地址映射：

在硬件支持下将逻辑地址转换为实际内存物理地址。

内存扩充：

  • 请求调入：部分装入，运行期间由操作系统在需要时装入所需部分。

  • 置换：内存空间不够时，可调出部分不活跃程序或数据到外存，将需要调入的部分放入内存。

（3）设备管理

为用户进程分配所需 I/O 设备，完成指定 I/O 操作，提高 CPU 和 I/O 设备利用率，方便用户使用 I/O 设备。

  • 缓冲管理：速度存在差异的两个设备之间通信时通常需要缓冲。

  • 设备分配：根据用户 I/O 请求，系统目前状况按照某种策略为用户分配所需的 I/O 设备。

  • 设备处理：通过设备驱动程序完成，实现 CPU 和设备控制器之间的通信。

（4）文件管理

文件存储空间管理：

  • 为文件分配必要的外存空间，提高外存利用率。

  • 提高外存存取速度。

  • 记录文件存储空间使用情况。

  • 对存储空间进行分配和回收。

目录管理：

  • 一个文件对应一个目录项，按名存取。

  • 文件共享。

  • 快速目录查询。

  • 文件读/写管理：根据用户需求，从外存读取数据，或将数据写入外存。

  • 文件保护：提高有效存取控制，防止未授权用户存取文件，以及以不正确的方式使用文件。

（5）提供用户和操作系统之间的接口

用户接口：

  • 联机用户接口：键盘命令

  • 脱机用户接口：批处理作业说明书

  • 图形用户接口：图形化操作界面

程序接口：系统调用

（6）系统安全

现代操作系统新增功能，如认证技术、密码技术、访问控制技术、反病毒技术等。

（7）网络功能和服务

实现网络通信和资源管理，提供网络服务。

（8）支持多媒体

如接纳控制，实时调度，多媒体文件存储等。

接口

用户接口（用户直接操作/操纵）

  • 联机用户接口：键盘命令、命令解释程序。

  • 脱机用户接口：批处理作业用户使用的由作业控制语言书写的作业说明书。

  • 图形用户接口：图形化操作界面。

程序接口（编程语言等）

即系统调用，为用户执行过程中访问资源而设置，是用户获取操作系统服务的唯一途径。

用户态 -> 核心态

主动方式：系统调用。

  • 用户态主动切换到内核态。

  • 申请使用操作系统提供给的服务程序完成工作。

被动方式：不可预知、可能存在。

  • 异常：用户程序发生了不可预知的意外事件而切换到相应内核程序，如缺页、死循环、除数为零等。

  • 外设中断：如磁盘完成 I/O。

核心态 -> 用户态

通过执行一个特权指令，将程序状态字（PSW）的标志位设置为“用户态”。

指令

简单来说，“指令”就是处理器（CPU）能识别、执行的最基本指令。

比如：加法指令就是让 CPU 进行加法运算。

特权指令

只能由操作系统使用、用户程序不能使用的指令。

示例：

  • 停机

  • 启动 I/O

  • 设置时钟

  • 允许/禁止中断

  • 修改程序状态字

  • 内存请除、改变存储映像图

  • …

非特权指令

用户程序可以使用的指令。

示例：

  • 读取时钟（例：状态栏查看时钟）

  • 访管指令（目态 -> 管态）

  • 控制转移（例：if…else、switch…case、for、while、goto）

  • 算术逻辑运算（例：+、-、*、/、%、^ 等）

  • 数据传送、存数、取数

中断和异常

中断的概念和作用

  • 1、当中断发生时，CPU立即进入核心态。

  • 2、当中断发生后，当前运行的进程将暂停运行，并由操作系统内核对中断进行处理。

  • 3、对于不同的中断信号，会进行不同的处理。

发生了中断，就意味着需要操作系统介入，开展管理工作。由于操作系统的管理工作（比如进程切换、分配 I/O 设备等）需要使用特权指令，因此 CPU 要从用户态转为核心态。

中断可以使 CPU 从用户态转为核心态，使操作系统获得计算机的控制权。

有了中断，才能实现多道程序并发执行。

中断的分类

内中断（也称异常、例外、陷入）

信号的来源：CPU 内部与当前执行的指令有关。

  • 自愿中断、指令中断（系统调用时使用的访管指令，又叫陷入指令，trap 指令）

  • 强迫中断 - 硬件故障（如：缺页）、软件中断（如：整数除 0）

外中断（狭义的中断）

信号的来源：CPU 外部与当前执行的指令无关。

  • 外设请求（如：I/O 操作完成发出的中断信号）

  • 人工干预（如：用户强行终止一个进程）

中断的另一种分类方式

内中断（内部异常）

信号来源：CPU 内部

  • 陷阱、陷入（trap）：有意为之的异常，如：系统调用。

  • 故障（fault）：由错误条件引起的，可能被故障处理程序修复，如：缺页。

  • 终止（abort）：不可恢复的致命错误造成的结果，终止处理程序不再将控制返回给引发终止的应用程序，如：整数除0。

外中断

信号来源：CPU 外部

  • I/O 中断请求

  • 人工干预

外中断的处理过程

  • Step1：执行完每个指令之后，CPU 都要检查当前是否有外部中断信号。

  • Step2：如果检测到外部中断信号，则需要保护被中断的 CPU 环境（如：程序状态字 PSW、程序计数器 PC、各种通用寄存器）

  • Step3：根据中断信号类型传入相应的中断处理程序。

  • Step4：恢复原进程的 CPU 环境并退出中断，返回原程序继续往下执行。

处理器状态

用程序状态字寄存器（PSW）中的某标志位来标识当前处理器处于什么状态。如：0为用户态，1为核心态。

核心态：管态/特权态/系统态

特权指令、非特权指令都能执行。

  • 运行操作系统程序，操作硬件。

  • 可执行包括特权指令在内的一切机器指令。

  • 程序不受硬件限制，可访问全部存储器和寄存器。

用户态：常态/目态

此时 CPU 只能执行非特权指令。

  • 运行用户程序。

  • 只能执行非特权指令。

  • 用户程序通常在此状态下执行。

  • 受硬件限制，访问有限的存储空间和指定寄存器。

程序

内核程序

操作系统的内核程序是系统的管理者，既可以执行特权指令，也可以执行非特权指令，运行在核心态。

应用程序

为了保证系统能安全运行，普通应用程序只能执行非特权指令，运行在用户态。

结构

无结构操作系统

主要任务：功能实现和系统高效，不考虑操作系统的结构性。

特点：

  • 由众多过程直接构成。

  • 各过程之间可任意相互调用。

  • 操作系统内部不存在任何结构，又称为整体系统结构。

缺点：

  • 庞大杂乱，缺乏清晰的程序结构。

  • 程序错误多，调试难、阅读难、理解难、维护难。

模块化操作系统

遵循 20 世纪 60 年代出现的结构化程序设计方法。

优点：

  • 提高了 OS 设计的正确性、可理解性和可维护性。

  • 增强了 OS 的可适用性。

  • 加速了 OS 的开发过程。

缺点：

  • 模块及接口划分较困难。

  • 未区别共享和独占资源。

  • 管理查理导致 OS 结构不够清晰。

分层式操作系统

  • 按分层式结构设计的基本准则，将 OS 划分为若干个层次。

  • 每一层都只能使用其底层提供的功能和服务。

  • 从硬件开始，在其上一层一层、自底向上增添相应功能对应的软件。

特点：

  • 每一步设计都建立在可靠的基础上，结构更加清晰。

  • 调试和验证更加容易，正确性更高。

微内核结构操作系统

  • 在OS 内核中只保留最基本的功能，将其他服务分离出去，形成 “客户/服务器” 模式。客户进程可通过内核向服务器进程发送请求，以取得 OS 的服务。

操作系统分为两个部分：微内核，多个服务器

  • 适用于分布式系统环境，支持多处理机运行的操作系统。

  • 实例：Windows、macOS、Linux

特点：

  • 足够小的内核：操作系统最基本的部分。

  • 基于 “客户/服务器” 模式。

  • 应用 “机制与策略分离” 原理。

优点：

  • 可移植性强。

  • 增强了系统的可靠性。

  • 融入了面向对象技术。

  • 提高了系统的可扩展性。

  • 提供了对分布式系统的支持。

缺点：系统效率低。

内核

内核是计算机上配置的底层软件，是操作系统最基本、最核心的部分，实现操作系统内核功能的程序就是内核程序。

大内核

将操作系统的主要功能模块都作为系统内核，运行在核心态。

  • 优点：高性能。

  • 缺点：内核代码庞大，结构混乱，难以维护。

微内核

只把最基本的功能保留在内核。

  • 优点：内核功能少，结构清晰，方便维护。

  • 缺点：需要频繁地在核心态和用户态之间切换，性能低。

类比

操作系统的体系结构问题与企业的管理问题很相似。

内核就是企业的管理层，负责一些重要的工作。只有管理层才能执行特权指令，普通员工只能执行非特权指令。用户态、核心态之间的切换相当于普通员工和管理层之间的工作交接。

  • 大内核：企业初创时体量不大，管理层的人会负责大部分的事情。优点是效率高；缺点是组织结构混乱，难以维护。

  • 微内核：随着企业体量越来越大，管理层只负责最核心的一些工作。优点是组织结构清晰，方便维护；缺点是效率低。

系统调用

操作系统作为用户和计算机硬件之间的接口，需要向上提供一些简单易用的服务。主要包括命令接口和程序接口。其中，程序接口由一组系统调用组成。

“系统调用”是操作系统提供给应用程序（程序员/编程人员）使用的接口，可以理解为一种可供应用程序调用的特殊函数，应用程序可以发出系统调用请求来获得操作系统的服务。

作用

应用程序通过系统调用请求操作系统的服务。系统中的各种共享资源都由操作系统统一掌管，因此在用户程序中，凡是与资源有关的操作（如：存储分配、I/O 操作、文件管理等），都必须通过系统调用的方向操作系统提出服务请求，由操作系统代为完成。这样可以保证系统的稳定性和安全性，防止用户进行非法操作。

按功能分类

系统调用相关处理涉及到对系统资源的管理、对进程的控制，这些功能需要执行一些特权指令才能完成，因此系统调用的相关处理需要在核心态下进行。

  • 设备管理

完成设备的 请求/释放/启动 等功能。

  • 文件管理

完成文件的 读/写/创建/删除 等功能。

  • 进程控制

完成进程的 创建/撤销/阻塞/唤醒 等功能。

  • 进程通信

完成进程之间的 消息传递/信号传递 等功能。

  • 内存管理

完成内存的 分配/回收 等功能。

系统调用与库函数的区别

不涉及系统调用的库函数，如：“取绝对值”函数

涉及系统调用的库函数，如：“创建一个新文件”函数

层级调用关系

裸机 -> 操作系统 -> 编程语言 -> 普通应用程序
  • 普通应用程序

可直接进行系统调用，也可以使用库函数。有的库涉及系统调用，有的不涉及。

  • 编程语言

向上提供库函数。有时会将系统调用封装成库函数，以隐藏系统调用的一些细节，使上层进行系统调用更加方便。

  • 操作系统

向上提供系统调用。
