ViewBinding

try to use layout id directly in you Activity / Fragment

after Gradle Version 6.5, if you want to use layout id directly, you should modify build.gradle file by yourself to enable this function.

PS: recommend to use ViewBinding or DataBinding

plugins {
    ...
    // add this
    id 'kotlin-android'
    id 'kotlin-android-extensions'
}

Navigation

navHostFragment: NavController Error

Error output:

java.lang.RuntimeException: Unable to start activity ComponentInfo{xxx.MainActivity}: java.lang.IllegalStateException: Activity xxx.MainActivity@xxx does not have a NavController set on xxx

to Solve this problem, try this code in your MainActivity or any where you want to use NavController:

  • before
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // set up the default AppBar to auto change the correct label
        setupActionBarWithNavController(findNavController(R.id.navHostFragment))
    }
}
  • after
class MainActivity : AppCompatActivity() {
    private lateinit var navController: NavController

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val navHostFragment = supportFragmentManager.findFragmentById(R.id.navHostFragment) as NavHostFragment
        navController = navHostFragment.navController

        // set up the default AppBar to auto change the correct label
        setupActionBarWithNavController(navController)
    }
}

if you don't want to add code at your Dev File, Try this

Change the FragmentContainerView to fragment

  • before
<androidx.fragment.app.FragmentContainerView
        android:id="@+id/navHostFragment"
        android:name="androidx.navigation.fragment.NavHostFragment"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:defaultNavHost="true"
        app:navGraph="@navigation/nav" />
  • after
<fragment
        android:id="@+id/navHostFragment"
        android:name="androidx.navigation.fragment.NavHostFragment"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:defaultNavHost="true"
        app:navGraph="@navigation/nav" />

Cannot resolve symbol ‚Äòcom.xxx.xxxFragmentArgs‚Äô in XML

if you are using DataBinding with Navigation, and try to pass value between Activity / Fragment in XML like the code below:

<layout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools">
    <data>
        <variable
            name="args"
            type="com.xxx.xxxFragmentArgs" />
    </data>

    <androidx.constraintlayout.widget.ConstraintLayout
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        tools:context=".xxxFragment">

        ...

    </androidx.constraintlayout.widget.ConstraintLayout>

</layout>

you can try add this code in your build.gradle

sourceSets {
        main {
            kotlin {
                srcDirs += 'build/generated/source/navigation-args'
            }
        }
    }

or Java Version:

sourceSets {
        main {
            java {
                srcDirs += 'build/generated/source/navigation-args'
            }
        }
    }

Room

Cannot create an instance of xxx.xxxViewModel

if you don't want to use Hilt and try to create an ViewModel use this code below:

private val mViewModel: MyViewModel by viewModels()

you should add this two line into your build.gradle.

plugins {
    ...
    
    // Add
    id 'kotlin-kapt'
}

...

dependencies {

    implementation 'androidx.core:core-ktx:x.x.x'
    implementation 'androidx.appcompat:appcompat:x.x.x'
    ...

    // Add
    kapt "androidx.room:room-compiler:2.4.2"
}

If you are using Hilt, ensure your activity/fragment is having @AndroidEntryPoint annotation

Make your constructor public.

Make sure your ViewModel has constructor with only one parameter i.e. Application.

example:

Dagger Hilt

Unresolved reference: ApplicationComponent

After Dagger-Version: 2.31, ApplicationComponent has been removed by Google

If you want to use the newer version, use SingletonComponent instead.

  • before
@InstallIn(ApplicationComponent::class)
@Module
object NetworkModule {

    ...
}
  • after
@InstallIn(SingletonComponent::class)
@Module
object NetworkModule {

    ...
}
